title: Fade Up

---

The `FadeUp` component animates elements as they scroll into view. It uses the `Framer Motion` library under the hood to provide smooth transitions.

---

## Installation

You'll need to install the `framer-motion` package:

### Using NPM

```bash
npm install framer-motion
```

### Using PNPM

```bash
pnpm install framer-motion
```

### Using Yarn

```bash
yarn add framer-motion
```

---

## Usage

The FadeUp component wraps any element and animates it when it becomes visible in the viewport.

### Basic Example

			<h1>Hello World</h1>
		</FadeUp>

}`

### Multiple Elements

You can wrap multiple elements individually for staggered animations:

```tsx
<FadeUp><h2>First Item</h2></FadeUp>
<FadeUp><h2>Second Item</h2></FadeUp>
<FadeUp><h2>Third Item</h2></FadeUp>
```

## Props

The `FadeUp` component accepts a few optional props to customize the animation.

`delay`

- Type: `number`
- Default: `0`
- Delays the animation in seconds. Useful when staggering multiple elements.

`duration`

- Type: `number`
- Default: `0.4`
- Controls how long the animation lasts.

`once`

- Type: `boolean`
- Default: `true`
- If true, the animation will only trigger the first time the element enters the viewport.

## Best Practices

- ✅ Use once for content that should animate in only once (e.g., hero sections).
- ✅ Use delay with lists to create staggered animations.
- ❌ Don’t overuse fade-up on large blocks of content — it can feel distracting.
